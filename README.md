# Aetherys Orchestrator
![Aetherys Orchestrator Workspace](https://images.unsplash.com/photo-1678097393903-881250b9858a?q=80&w=2574&auto=format&fit=crop&ixlib=rb-4.0.3&ixid=M3wxMjA3fDB8MHxwaG90by1wYWdlfHx8fGVufDB8fHx8fA%3D%3D 'Aetherys Orchestrator Triple-Pane Interface')

## ‚ú® Key Features
- **Triple-Pane Workspace**: A fluid layout with a Command Center sidebar, a generative Nexus Chat interface, and a dynamic Artifact Foundry panel.
- **Real-Time Conversational AI**: Streaming chat with rich markdown rendering, multi-modal support (STT/TTS), and intelligent tool usage.
- **Artifact Rendering**: A dedicated panel for code previews, live HTML rendering, SVG, and interactive canvas artifacts generated by the AI.
- **MCP Tool Integration**: Dynamically connect to remote Model Context Protocol (MCP) servers for extensible, real-time tool discovery and execution.
- **Cloudflare-Native Tooling**: Simulated tools for deploying Workers, managing R2 files, and querying D1 databases, demonstrating deep integration with the Cloudflare ecosystem.
- **Advanced Session Management**: Persistent chat history via Durable Objects, with intuitive controls for creating, renaming, and deleting sessions.
- **Cyberpunk UI**: A stunning interface featuring glassmorphism effects, smooth animations (Framer Motion), and a fully responsive design.
- **File & Knowledge Management**: Upload files to a session's context, perform semantic searches, and manage your knowledge base through a sleek interface.
> **Note**: The AI capabilities of this application are rate-limited on shared servers. For production use, it is highly recommended to configure your own Cloudflare AI Gateway for optimal performance and reliability.
## üöÄ Quick Start
1.  **Clone & Install**:
    ```bash
    git clone <your-repo-url> aetherys-orchestrator
    cd aetherys-orchestrator
    bun install
    ```
2.  **Configure Environment**:
    Open `wrangler.jsonc` and update the `vars` section with your Cloudflare AI Gateway credentials.
    ```json
    "vars": {
      "CF_AI_BASE_URL": "https://gateway.ai.cloudflare.com/v1/{YOUR_ACCOUNT_ID}/{YOUR_GATEWAY_ID}/openai",
      "CF_AI_API_KEY": "your-cloudflare-api-key"
    }
    ```
3.  **Run Development Server**:
    ```bash
    bun run dev
    ```
    The application will be available at `http://localhost:3000`.
4.  **Deploy to Cloudflare**:
    ```bash
    bun run deploy
    ```

## üîß Configuration
### Environment Variables
All backend configurations are managed within `wrangler.jsonc`. For production, use Wrangler secrets.
| Variable           | Description                                                              | Required |
| :----------------- | :----------------------------------------------------------------------- | :------- |
| `CF_AI_BASE_URL`   | Your Cloudflare AI Gateway endpoint URL.                                 | **Yes**  |
| `CF_AI_API_KEY`    | Your Cloudflare API key for the AI Gateway.                              | **Yes**  |
| `SERPAPI_KEY`      | (Optional) API key for SerpApi to enable live web search.                | No       |
| `OPENROUTER_API_KEY`| (Optional) API key for OpenRouter integration.                           | No       |
To set a secret for production:
```bash
wrangler secret put CF_AI_API_KEY
```
### In-App Settings
- **AI Models**: Configure the default model for new conversations.
- **MCP Servers**: Dynamically add or remove connections to remote MCP tool servers.
- **API Config**: View your configured AI Gateway endpoint (read-only for security).
- **Preferences**: Switch between light and dark themes.
## üß∞ Available Tools
The AI can leverage the following tools, some of which are simulated to demonstrate functionality without requiring live bindings.
| Tool Name               | Description                                                              | Type       |
| :---------------------- | :----------------------------------------------------------------------- | :--------- |
| `get_weather`           | Retrieves current weather for a specified location.                      | Simulated  |
| `web_search`            | Searches the web with SerpApi or fetches content from a URL.             | Live (API) |
| `deploy_worker`         | Deploys a Cloudflare Worker and generates a code artifact.               | Simulated  |
| `manage_r2_file`        | Uploads, retrieves, or deletes files from a session's R2 context.        | Simulated  |
| `search_knowledge_base` | Performs a semantic search on files within the session's context.        | Simulated  |
| *(MCP Tools)*          | Any tools discovered dynamically from connected MCP servers.             | Live (MCP) |
## üí° Troubleshooting
<details>
  <summary><strong>"Speech recognition is not supported" error</strong></summary>
  <p>This feature relies on the Web Speech API, which is not available in all browsers (e.g., Firefox). For the best experience, please use a Chromium-based browser like Google Chrome or Microsoft Edge.</p>
</details>
<details>
  <summary><strong>Microphone access is denied</strong></summary>
  <p>If you see an error about microphone permissions, you need to grant access in your browser's site settings. Click the lock icon in the address bar and ensure "Microphone" is set to "Allow" for this site.</p>
</details>
<details>
  <summary><strong>AI responses are slow or fail</strong></summary>
  <p>This can be due to rate limits on the shared AI Gateway. For consistent performance, configure your own Cloudflare AI Gateway in `wrangler.jsonc`. If the issue persists, check the status of your configured AI model provider.</p>
</details>
<details>
  <summary><strong>Failed to add/remove MCP Server</strong></summary>
  <p>Ensure the Worker is running and accessible. Check the Worker logs (`wrangler tail`) for any errors related to Durable Object storage or network requests.</p>
</details>
